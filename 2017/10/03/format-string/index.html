format string

##0x01成因
 因为*printf函数族的特性造成。

printf函数原型printf FORMAT [ARGUMENT]…
由FORMAT格式串决定输出参数个数和格式。

如果格式串中的参数个数多于函数传参个数就会造成越界访问内存。

##0x02利用
 通过控制格式串内容，利用*printf函数，达到任意内存读写的目的。

  • 利用条件
    • printf格式串位置可控或间接可控
  • 利用方法
    • 泄漏内存信息 配合其它漏洞
    • 更改关键局部变量 或关键函数got表 达到题目指定要求
    • 存在溢出和cananry时可以更改stack_check_fail函数 直接rop
    • 更改stdout文件指针内部结构控制函数执行流
    • 更改hook函数
    • 利用ebp中保持的栈指针 在栈上利用格式串写rop
  • payload构造
    • 判断offset 通过不断改变n 直至找 0x41414141 即为格式串偏移 payload = “AAAA”+“%”+”n”+”$p”
    • dump内存 利用切片操作取出中间内容 反复操作就可以dump内存payload = “AAAAA%n$sBBBB”+addr
    • 任意地址写 addr -> add2payload = “addr”+”%”+str(add2-len(addr)-1)+”c”+”%”+str(offset)+””$n”

##0x03类库使用

  • 常见类库 pwntools libformatstr
    • 功能：自动化探测偏移 根据提供的字典进行payload构造
    • 问题：有0x00字节截断问题
